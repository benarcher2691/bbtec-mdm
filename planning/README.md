**Guidelines:**
- Keep `PLAN-*.md` focused on FUTURE work only (no history)
- Archive completed planning docs to `archive/YYYY-MM-session/`

- Create new `PLAN-YYYY-MM-DD.md` for major milestones

**When to Archive:**
- Session planning docs when work completes
- Implementation plans when features ship
- Research artifacts after decisions made
- Technical reports after issues resolved

**When to Promote to docs/:**
- Patterns that apply long-term (e.g., authentication, testing)
- Operational guides for production use
- Architecture decisions that affect future development
- Build and deployment procedures
